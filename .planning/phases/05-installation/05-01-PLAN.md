  <action>
Create install.sh with the following structure:

1. **Header and shebang**: `#!/usr/bin/env bash`
2. **Set strict mode**: `set -euo pipefail`
3. **Define variables**:
   - `INSTALL_DIR="$HOME/.shell-explainer"` (per D-04)
   - `PLUGIN_FILE="$INSTALL_DIR/errormux.plugin.zsh"` (per D-05)
   - `SOURCE_LINE="source ~/.shell-explainer/errormux.plugin.zsh"` (per D-06)
   - `REPO_URL` placeholder (user will update with actual URL)
4. **Pre-flight checks**:
   - Check `command -v uv` exists; if not, print error message and exit 1 (per D-09)
   - Check `command -v git` exists
5. **Idempotent clone** (per D-03):
   - If `$INSTALL_DIR` exists and contains `.git`, pull latest: `git -C "$INSTALL_DIR" pull`
   - Otherwise, clone fresh: `git clone "$REPO_URL" "$INSTALL_DIR"`
6. **Error handling**: Print colored messages (green for success, red for errors)

Example skeleton:
```bash
#!/usr/bin/env bash
set -euo pipefail

INSTALL_DIR="$HOME/.shell-explainer"
PLUGIN_FILE="$INSTALL_DIR/errormux.plugin.zsh"
SOURCE_LINE="source ~/.shell-explainer/errormux.plugin.zsh"
REPO_URL="https://github.com/YOUR_USERNAME/errormux.git"

# Colors for output
GREEN='\033[0;32m'
RED='\033[0;31m'
NC='\033[0m'

info() { echo -e "${GREEN}[errormux]${NC} $1"; }
error() { echo -e "${RED}[errormux error]${NC} $1"; exit 1; }

# Pre-flight checks
command -v uv >/dev/null || error "uv is required but not installed. See: https://docs.astral.sh/uv/"
command -v git >/dev/null || error "git is required but not installed."

# Clone or update repository
if [[ -d "$INSTALL_DIR/.git" ]]; then
    info "Updating existing installation..."
    git -C "$INSTALL_DIR" pull
else
    info "Cloning errormux to $INSTALL_DIR..."
    rm -rf "$INSTALL_DIR"
    git clone "$REPO_URL" "$INSTALL_DIR"
fi
```

DO NOT install the package yet (that's Task 3). Just set up the clone logic.
