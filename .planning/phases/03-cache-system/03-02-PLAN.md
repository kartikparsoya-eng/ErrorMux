## Task 03-02-01: Add cache check to CLI explain()

<read_first>
- `src/errormux/cli.py` — current explain() implementation
- `src/errormux/cache.py` — cache module from plan 01
- `.planning/phases/03-cache-system/03-CONTEXT.md` — D-04 cache before prompt build
</read_first>

<action>
Modify `src/errormux/cli.py`:

1. Add import: `from errormux.cache import make_cache_key, cache_get, cache_set`

2. In `explain()` function, after `read_context()` succeeds:
   - Generate cache key: `cache_key = make_cache_key(cmd, stderr)`
   - Check cache: `cached_response = cache_get(cache_key)`
   - If `cached_response` is not None:
     - Parse cached response: split on `\n` to get WHY and FIX lines
     - Print with Rich formatting (same as fresh response)
     - `return` early (skip Ollama call)

3. After successful `parse_response()` (when `why` and `fix` are extracted):
   - Cache the response: `cache_set(cache_key, f"WHY: {why}\nFIX: {fix}")`
   - Continue with existing print logic

4. The flow becomes:
   ```
   read_context() → cache_key → cache_get()
   ↓ hit: print cached, return
   ↓ miss: build_user_prompt → chat_with_ollama → parse_response → cache_set → print
   ```
</action>
