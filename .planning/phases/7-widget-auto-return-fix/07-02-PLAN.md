## Test Cases

### Test 1: Basic Auto-Return
1. Run a failing command: `ls /nonexistent`
2. Press `Ctrl+X` then `?`
3. **Expected**: See WHY/FIX explanation, then prompt returns to ready state WITHOUT needing to press Enter

### Test 2: Pending Input Preservation
1. Type a partial command but don't press Enter: `echo hel`
2. Press `Ctrl+X` then `?`
3. **Expected**: See explanation, then `echo hel` is still in your input buffer, ready to continue typing

### Test 3: No Flicker
1. Run a failing command: `cat /missing`
2. Press `Ctrl+X` then `?`
3. **Expected**: Explanation displays cleanly, prompt redraws without visual artifacts

### Test 4: Skip-List Message
1. Run `false` (exit 1, common command)
2. Press `Ctrl+X` then `?`
3. **Expected**: See "nothing to explain" message, prompt returns cleanly

1. Stop Ollama: `ollama stop gemma3:4b` or `pkill ollama`
2. Run a failing command: `ls /nonexistent`
3. Press `Ctrl+X` then `?`
4. **Expected**: See "[explainer offline]" message, prompt returns cleanly
