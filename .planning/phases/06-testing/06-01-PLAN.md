# Phase 06 Plan 01: Add pytest-cov Configuration

**Objective:** Configure pytest-cov in pyproject.toml with 80% coverage threshold enforcement for automated test coverage measurement.

## Context

Phase 6 requires 80% code coverage (D-01). pytest-cov must be added as a dev dependency and configured to:
- Fail builds when coverage drops below 80%
- Show uncovered lines in terminal output
- Exclude untestable code patterns

---

## Tasks

### Task 1: Add pytest-cov dependency

<read_first>
- pyproject.toml
</read_first>

<action>
Update pyproject.toml to add pytest-cov to dev dependencies:

```toml
[dependency-groups]
dev = [
    "pytest>=8.0",
    "pytest-cov>=5.0",
]
```

Replace the deprecated `[tool.uv]` dev-dependencies section with `[dependency-groups]` format.
</action>

<acceptance_criteria>
- `pyproject.toml` contains `[dependency-groups]` section with `pytest-cov>=5.0`
- `pyproject.toml` does NOT contain `[tool.uv]` dev-dependencies (deprecated)
- `uv sync` completes successfully
- `uv run pytest --cov=errormux --help` shows pytest-cov help output
</acceptance_criteria>

---

### Task 2: Configure pytest coverage settings

<read_first>
- pyproject.toml
</read_first>

<action>
Add pytest configuration section to pyproject.toml:

```toml
[tool.pytest.ini_options]
addopts = "-v --cov=errormux --cov-fail-under=80 --cov-report=term-missing"
testpaths = ["tests"]

[tool.coverage.run]
source = ["src/errormux"]
branch = false

[tool.coverage.report]
exclude_lines = [
    "pragma: no cover",
    "if TYPE_CHECKING:",
    "raise NotImplementedError",
]
```
</action>

<acceptance_criteria>
- `pyproject.toml` contains `[tool.pytest.ini_options]` section
- `addopts` includes `--cov=errormux --cov-fail-under=80`
- `pyproject.toml` contains `[tool.coverage.run]` section with `source = ["src/errormux"]`
- `pyproject.toml` contains `[tool.coverage.report]` section with `exclude_lines`
</acceptance_criteria>
